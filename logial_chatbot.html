Introduction

In this project you will be building a chatbot for placing an order for "5100 Pizza Place". Your chatbot (refered to as PizzaBot from here on), will use a backend Knowledge Base that will check for missing constraints, and ask user for information to guide them through the process of ordering their pizza.

For this project you will need to install Pyke which is a Python based Knowledge Engine, inspired by Prolog. You will encode your knowledgebase in Pyke. Once you download Pyke, please follow the instructions in the readme file to build and install it. You can confirm installation by running on of the examples from the following directory: 

pyke/examples

Evaluation: You will be evaluated on the complexity of your system and complexity of your knowledgebase.

Question 2 (10 points): Relationships between facts

In order to place a pizza order, what types of dependencies need to be present between different facts?

  • Can you decide toppings, without picking sauce first?
  • Can you pick your sauce without picking the crust/base?
  • What are the minimum relations that need to be satisfied in order to complete a pizza order?
For this part of your project you will mostly work with the following files: 
            relation.krb
            Readme.txt
You will be implementing this using forward chaining. Please mention in your readme file how your approach would change if you used backward chaining.

Question 3 (20 points): Asking Questions and Understanding Answers: KB to Text Mapping

In order to place a pizza order, you will need to write rules that map knowledgebase facts into english language. This needs to be updated in the following files: 

        kb2text.py
        readme.txt

How would you design your system to ask the relevant questions to the user? Remember, your questions will require answers from the users. These answers will provide assertions or true values to your knowledgebase facts. You can assume that the user will enter simplistic English. e.g. 
        PizzaBot: Please pick your toppings: Onion, Tomato, Peppers
        User: Onion Tomato
        PizzaBot: Toppings selected
        PizzaBot: Do you want extra cheese? Yes or No?
        User: No
Notice that at each turn, PizzaBot is guiding the user regarding what answer it is expecting from the user. You will need to build your own simplistic English based on your fact based knowledgebase. You will also need to assert facts dynamically based on what the user enters/selects for their order. You will need to get familiar with Pyke Documentation to understand how to dynamically assert facts.
